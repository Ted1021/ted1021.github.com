간편한 스케줄 등록

자동으로 길찾기

저장된 약속정보와 현재위치를 바탕으로 대중교통 길찾기를 수행합니다. 더이상 번거롭게 다른 길찾기앱을 켜지마세요!

출발시간 관리는 덤

어떤 이동수단을 이용할지, 이동하는데 얼마나 시간이 걸리는지를 파악해 사용자의 출발시간을 자동으로 계산해 줍니다.

Main Features

Calendar

특정 날짜에 저장된 스케줄의 갯수와 그 컨텐츠를 직관적으로 확인 할 수 있도록 UI 구성하였습니다.

Today Schedule

오늘의 스케줄 정보와 이동정보를 한눈에 파악하고 '미리도착', '원하는 이동경로 선택' 과 같은 메뉴들을 통해 좀 더 유연하게 스케줄을 관리 할 수 있습니다.

Notification

알람 시점, 현재위치, 대중교통정보를 종합해 사용자에게 적절한 알람을 제공합니다.

Technologies

다음의 기술들을 이용해 서버와 데이터베이스, 네트워크를 구성하고 앱 서비스를 구현해 보았습니다.

Realm

Realm 에서 제공하는 Live Data, Object Listener, RealmAdapter 등을 이용해 스케줄과 교통정보, 알림정보 등을 효율적을 관리

NodeJS

클라우드 서버인 Digital Ocean 에서 Ubuntu 와 NodeJS 로 서버를 구성하고, Odsay API 서버와 Android 클라이언트 사이를 연결하는 징검다리 구축

Retrofit2

Retrofit2 를 이용해 서버와 통신하고, 대중교통 이동정보를 JSON 형식으로 교환

Odsay API

위치 데이터를 기반으로 대중교통정보, 환승정보, 길찾기정보 등을 검색하고 어플리케이션의 기획의도에 맞게 데이터를 가공

Event Bus

Event Bus 를 이용해 어플리케이션 내의 다양한 클래스들 사이에서 빠르고 정확한 데이터 교환 로직을 구성

Service

어플리케이션을 사용하고 있지 않더라도 Background 에서 사용자의 위치를 주기적으로 갱신하고, 자동으로 알람시점을 업데이트

Boost up points

"부스트캠프 안드로이드 2기 과정" 을 수료하기 전과 후를 되짚어 보면서

개인적으로 어떤 성장을 이루게 되었는지 정리 해보았습니다.

"하나의 데이터, 여러 UI 그리고 Transaction..."

앱서비스의 구성상, Activity, Fragment, RecyclerView, Service 등 정말 다양한 요소들로부터 스케줄 데이터가 변경될 수 있었는데 이때마다 발생하는 트랜잭션 충돌에 대한 해결책이 필요했습니다.

Realm

Database 스케줄, 일정데이터 관리

"쉬운 기능, 어려웠던 첫걸음"

부스트캠프에 입과하기 전까지 한번도 서버나 자바스크립트 관련 학습을 한적이 없었기 때문에 과정을 시작할 때 가장 큰 걱정이 되었던 것은 바로 "서버 구현" 이었습니다.

Node JS

Server 서버, 클라이언트 통신

"많은것을 가능케해준 서비스"

올해 초부터 안드로이드를 본격적으로 시작하게 되었지만 아직 한번도 "서비스" 컴포넌트를 사용해본 적이 없었습니다.

단순히 서버로부터 정제된 데이터를 가져와 사용자에게 보여주는 것이 전부였는데, 이번 서비스를 개발하게 되면서 안드로이드의 활용 영역을 넓히게 된 계기가 되었습니다.

Service

Thread 위치정보 및 알림정보 업데이트

"App 이 밋밋하다!!"

저희 튜터님으로부터 들었던 첫번째 피드백입니다. 디자인적인 부분은 거의 문외한에 가까웠지만 거듭된 튜터링을 통해 생각보다 많은 요소에 머테리얼 디자인이 필요하다는 것을 알게 되었습니다.

Elevation, 색상 등 다양한 디자인 규격에 관심을 가지게 되었고 이는 앱을 좀 더 풍성하게 만드는 계기가 되었습니다.

Additional Features
